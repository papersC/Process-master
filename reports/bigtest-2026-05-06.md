### Batch 4 — security probes (XSS / SQLi / CSP)
**SQL injection:** clean. All 41 `ExecuteSqlRawAsync` calls live inside
`Program.cs` bootstrap/seeder paths with hard-coded SQL — no user input.

**XSS sinks:** 3 stored-XSS vectors found in `Views/Shared/_Layout.cshtml`,
all in code that interpolates server-side data into `innerHTML`:

| Spot | Sink | Source | Severity |
|------|------|--------|----------|
| Quick search results (~L1279) | `${item.url}` in `href`, `${item.name}`, `${item.description}`, `${item.type}` | `/api/search` — names + descriptions are user-input via Create forms | 🔴 Critical |
| AI suggestions (~L1676) | `${s}` in button innerHTML + inline `onclick` reading via `.textContent` | AI service response | 🟠 High |
| Notifications dropdown (~L1602) | `${title}`, `${message}`, `${n.actionUrl}` in `href`, `${n.id}` in inline `onclick` | Any user/service that sends a notification | 🔴 Critical |

**Fix:** Added shared helpers `spEscapeHtml(v)` and `spSafeUrl(u)` at the top
of the layout's main script block. `spSafeUrl` blocks `javascript:` / `data:`
/ `vbscript:` schemes (returns `#`). All three sinks now run server data through
the appropriate helper. Notifications + suggestions also moved off inline
`onclick` to delegated handlers (cleaner under CSP without `'unsafe-inline'`).

**Verified live:**
- `spEscapeHtml('<img src=x onerror=alert(1)>')` → `&lt;img src=x onerror=alert(1)&gt;` ✅
- `spSafeUrl('javascript:alert(1)')` → `#` ✅
- `spSafeUrl('/Improvements/Details/abc')` → `/Improvements/Details/abc` ✅
- Dashboard loads with zero console errors ✅

**Auto-fixed:** 3
**Needs review:** 0
**Tests:** ran 4 helper-behavior + smoke checks · passed 4 · failed 0

## Top critical issues (now fixed)

1. **Stored XSS in quick-search dropdown** (`_Layout.cshtml` ~L1279). Search
   API result `name` / `description` / `url` interpolated raw into innerHTML.
   Any user with Create permission could store `<img onerror=…>` in a
   Process or Service name and execute on every searcher's page.
   *Fix:* `spEscapeHtml` + `spSafeUrl` helpers (commit `a2e0e59`).

2. **Stored XSS in notifications dropdown** (`_Layout.cshtml` ~L1602). Same
   pattern — `title`, `message`, `actionUrl`, `id` from server. Any
   notification trigger (workflow, scheduler, AI) could inject HTML.
   *Fix:* same helpers + delegated click handler instead of inline onclick
   (commit `a2e0e59`).
